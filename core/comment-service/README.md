# Комментарии к событиям

Функциональность вынесена в отдельный микросервис `comment-service` и позволяет пользователям оставлять комментарии к событиям с системой модерации.

## 🚀 API Endpoints

### Private API (для авторизованных пользователей)

#### Создать комментарий
```
POST /users/{userId}/comments
```

**Тело запроса:**
```json
{
  "text": "Текст комментария",
  "eventId": 1
}
```

**Ответ:**
```json
{
  "id": 1,
  "text": "Текст комментария",
  "eventId": 1,
  "authorId": 123,
  "authorName": "Имя автора",
  "status": "PENDING",
  "created": "2023-10-01T12:00:00",
  "updated": null
}
```

#### Редактировать комментарий
```
PATCH /users/{userId}/comments/{commentId}
```

**Тело запроса:**
```json
{
  "text": "Обновленный текст комментария"
}
```

#### Удалить комментарий
```
DELETE /users/{userId}/comments/{commentId}
```

#### Получить мои комментарии
```
GET /users/{userId}/comments
```

### Public API (для всех пользователей)

#### Получить комментарии события
```
GET /events/{eventId}/comments
```

**Ответ:** Список опубликованных комментариев

#### Получить конкретный комментарий
```
GET /comments/{commentId}
```

### Admin API (для администраторов)

#### Модерировать комментарий
```
PATCH /admin/comments/{commentId}
```

**Тело запроса:**
```json
{
  "status": "PUBLISHED"
}
```
Доступные статусы: `PUBLISHED`, `REJECTED`

#### Удалить комментарий
```
DELETE /admin/comments/{commentId}
```

#### Поиск комментариев
```
GET /admin/comments?status=PENDING&eventId=1&userId=123
```

**Параметры:**
- `status` - фильтр по статусу
- `eventId` - фильтр по событию
- `userId` - фильтр по автору

## 📊 Статусы комментариев

- **PENDING** - на модерации (создается автоматически)
- **PUBLISHED** - одобрен модератором
- **REJECTED** - отклонен модератором
- **DELETED** - удален

## 🔄 Workflow статусов

```
PENDING → PUBLISHED  (админ одобрил)
PENDING → REJECTED   (админ отклонил)  
PENDING → DELETED    (автор удалил)
PUBLISHED → DELETED  (автор или админ удалил)
REJECTED → DELETED   (автор удалил)
```

## 🛡️ Правила доступа

- **Пользователи** могут редактировать только свои комментарии со статусом `PENDING`
- **Администраторы** могут модерировать любые комментарии и принудительно удалять
- **Публичный API** показывает только комментарии со статусом `PUBLISHED`

## 💾 Модель данных

| Поле | Тип | Описание |
|------|-----|----------|
| id | Long | Уникальный идентификатор |
| text | String | Текст комментария (1-2000 символов) |
| eventId | Long | Ссылка на событие |
| authorId | Long | Ссылка на автора |
| status | CommentStatus | Статус комментария |
| created | LocalDateTime | Дата создания |
| updated | LocalDateTime | Дата последнего обновления |
